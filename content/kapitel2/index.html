    Kapitel 2 – Unternehmenslizenzen¶

    Was du heute lernst¶

    • User-Lizenzen und Gerätelizenzen unterscheiden
    • Named User License (NUL) erklären und einordnen
    • Verstehen, wie die KMS-Aktivierung in Unternehmen funktioniert
    • CAL-Lizenzen für Windows Server kennen und anwenden
    • Lizenzbedarfe aus Unternehmensszenarien ableiten

    2.1 User-Lizenzen vs. Gerätelizenzen¶

    Im Unternehmensumfeld gibt es zwei grundlegende Konzepte, wie Software lizenziert werden kann:

    User-Lizenz (Nutzer-Lizenz)¶

    Eine User-Lizenz ist an eine bestimmte Person gebunden. Der Nutzer darf die Software auf mehreren Geräten installieren – aber immer nur er selbst.

    Beispiel: Eine Office-365-Lizenz für „Anna Müller". Anna darf Office auf ihrem Büro-PC, ihrem Laptop und ihrem Heimrechner nutzen – andere Mitarbeiter jedoch nicht.

    Geräte-Lizenz (Device License)¶

    Eine Geräte-Lizenz ist an ein bestimmtes Gerät gebunden. Auf diesem Gerät dürfen beliebig viele Nutzer die Software verwenden – aber die Lizenz gilt nur für dieses eine Gerät.

    Beispiel: Ein Kassensystem mit einer Gerätelizenz. Verschiedene Kassierer nutzen dasselbe Gerät – die Lizenz bleibt am Gerät.

    Wann welches Modell?

    User-Lizenz eignet sich, wenn Mitarbeiter von mehreren Geräten aus arbeiten (z. B. Büro + Homeoffice).

    Gerätelizenz ist sinnvoll, wenn viele Personen dasselbe Gerät teilen (z. B. Schichtbetrieb, Kassensysteme).

    2.2 Named User License (NUL)¶

    Eine Named User License (NUL) ist eine spezifische Form der User-Lizenz. Sie ist namentlich einer konkreten Person zugeordnet und kann nicht auf andere übertragen werden, ohne einen formalen Verwaltungsprozess.

    Typische Einsatzbereiche¶

    • SAP-Systeme (SAP Professional User License)
    • Adobe Creative Cloud (pro Person)
    • Spezialsoftware in der Buchhaltung oder im ERP-System

    Häufiger Fehler

    Eine NUL darf nicht geteilt werden. Wenn zwei Mitarbeiter sich einen Login teilen, um Lizenzkosten zu sparen, ist das eine Lizenzverletzung – auch wenn die Software technisch funktioniert.

    2.3 KMS-Aktivierung¶

    KMS steht für Key Management Service. Es ist eine Technologie von Microsoft, mit der große Unternehmen Windows und Office zentral aktivieren können – ohne dass jedes Gerät einzeln mit Microsoft kommunizieren muss.

    Wie funktioniert KMS?¶

    flowchart LR
    A[Client-PC] -->|Aktivierungsanfrage| B[KMS-Server\nim Firmennetz]
    B -->|Aktivierungsbestätigung| A
    B -->|Einmalige Registrierung| C[Microsoft-Server]
    1. Das Unternehmen richtet einen KMS-Server im internen Netzwerk ein
    2. Jeder PC im Netzwerk aktiviert sich beim KMS-Server (nicht bei Microsoft direkt)
    3. Der KMS-Server kommuniziert nur einmalig mit Microsoft
    4. Die Aktivierung der Clients muss alle 180 Tage erneuert werden – solange der KMS-Server erreichbar ist, geschieht das automatisch

    Voraussetzungen¶

    • Mindestens 25 Clients (Windows) oder 5 Clients (Office) müssen den KMS-Server kontaktieren, damit er aktiv wird
    • Der PC muss sich mindestens alle 180 Tage im Firmennetz einloggen (oder per VPN)

    Praktische Relevanz

    Im Unternehmensalltag wirst du oft KMS-aktivierte Windows-Installationen sehen. Erkennbar daran, dass die Aktivierung im Firmennetz automatisch klappt, aber ohne Netzwerkanschluss eine Warnung erscheint.

    2.4 CAL-Lizenzen für Windows Server¶

    CAL steht für Client Access License – eine Zugriffslizenz für Windows Server.

    Wenn Mitarbeiter auf einen Windows Server zugreifen (z. B. für Dateifreigaben, Drucker, Active Directory), benötigt jeder Zugreifende eine CAL – zusätzlich zur Server-Lizenz selbst.

    Wichtig

    Die Windows-Server-Lizenz erlaubt den Betrieb des Servers. Die CAL erlaubt den Zugriff auf den Server. Beides wird benötigt!

    Die zwei CAL-Typen¶

    User-CAL¶

    Eine User-CAL ist an eine Person gebunden. Der Nutzer darf von beliebig vielen Geräten aus auf den Server zugreifen.

    Wann sinnvoll? Wenn Mitarbeiter von mehreren Geräten aus zugreifen (PC + Laptop + Terminal).

    Device-CAL¶

    Eine Device-CAL ist an ein Gerät gebunden. Vom diesem Gerät aus darf eine beliebige Anzahl von Nutzern auf den Server zugreifen.

    Wann sinnvoll? Wenn viele Mitarbeiter dasselbe Gerät nutzen (z. B. Schichtarbeit, Produktionsrechner).

    Entscheidungsdiagramm¶

    flowchart TD
    A[Wie greifen Nutzer auf den Server zu?] --> B{Jeder Nutzer hat\nsein eigenes Gerät?}
    B -->|Ja| C[User-CAL\npro Person]
    B -->|Nein| D{Mehrere Nutzer\nteilen Geräte?}
    D -->|Ja| E[Device-CAL\npro Gerät]
    D -->|Weiß nicht| F[Kosten vergleichen:\nWeniger Nutzer → User-CAL\nWeniger Geräte → Device-CAL]

    Aufgaben – Kapitel 2¶

    Grundlagen User-CAL oder Device-CAL?

    Ein Unternehmen hat einen Windows Server für Dateifreigaben. Es gibt 20 Mitarbeiter und 15 Arbeitsplatz-PCs.

    Frage: Welcher CAL-Typ ist in diesem Fall günstiger – User-CAL oder Device-CAL? Rechne kurz nach und erkläre warum.

    Tipp

    User-CAL = eine CAL pro Person. Device-CAL = eine CAL pro Gerät. Überlege: Welche Zahl ist kleiner – Personen oder Geräte?

    Lösung

    Device-CAL ist günstiger, weil 15 Geräte < 20 Personen.

    • User-CAL: 20 Lizenzen (eine pro Mitarbeiter)
    • Device-CAL: 15 Lizenzen (eine pro PC)

    Das Unternehmen spart 5 Lizenzen durch Device-CALs.

    Hinweis: Voraussetzung ist, dass die Mitarbeiter nur von diesen 15 PCs aus auf den Server zugreifen.

    Grundlagen NUL – Regelverstoß erkennen

    Die „Müller Logistik GmbH" hat 10 Named-User-Lizenzen für ihre Lagerverwaltungssoftware.

    Da die Nachtschicht oft unterbesetzt ist, teilen sich zwei Nachtschicht-Mitarbeiter (Thomas und Kevin) einen Login-Account, um nicht noch eine Lizenz kaufen zu müssen.

    Fragen: 1. Liegt hier ein Lizenzverstoß vor? Begründe. 2. Wie könnte das Unternehmen das Problem legal lösen?

    Tipp

    Eine Named User License ist namentlich einer Person zugeordnet – der Name ist Programm.

    Lösung

    Ja, es liegt ein Lizenzverstoß vor.

    Eine Named User License (NUL) ist an eine konkrete Person gebunden. Das Teilen eines Accounts bedeutet, dass eine Lizenz von zwei Personen genutzt wird – das ist nicht erlaubt, auch wenn sie nie gleichzeitig aktiv sind.

    Legale Lösung: Die Firma muss eine zusätzliche Lizenz für den zweiten Nutzer kaufen. Alternativ: Wenn nie beide gleichzeitig arbeiten, könnte ein Concurrent-User-Modell (gleichzeitige Nutzer) günstiger sein – falls der Hersteller das anbietet.

    Vertiefung KMS-Aktivierung verstehen

    Erkläre die KMS-Aktivierung einem neuen Azubi, der bisher nur Heimrechner kennt.

    Dein Text sollte folgende Punkte abdecken:

    1. Was ist der Unterschied zur „normalen" Windows-Aktivierung bei einem Heimrechner?
    2. Warum macht KMS für Unternehmen Sinn?
    3. Was passiert, wenn ein Firmen-Laptop 6 Monate nicht mehr im Firmennetz war?
    Tipp

    Bei einer normalen Windows-Aktivierung zu Hause gibst du deinen Produktschlüssel ein und Windows aktiviert sich direkt bei Microsoft. Im Unternehmen läuft das über einen internen Server.

    Lösung

    1. Unterschied zur Heimaktivierung: Zu Hause gibt man einen einzigartigen Produktschlüssel ein, und Windows aktiviert sich direkt bei Microsoft über das Internet.

    Bei KMS im Unternehmen gibt es keinen individuellen Schlüssel pro PC. Stattdessen kontaktiert der PC einen internen KMS-Server im Firmennetz, der die Aktivierung stellvertretend übernimmt. Microsoft bekommt nur Kontakt zum Server, nicht zu jedem einzelnen PC.

    2. Warum KMS für Unternehmen? - Kein Aufwand mit einzelnen Produktschlüsseln für 500+ PCs - Zentrale Kontrolle und Protokollierung - Spart Zeit bei Neuinstallationen

    3. Laptop nach 6 Monaten: Die KMS-Aktivierung muss alle 180 Tage erneuert werden. Nach 6 Monaten ohne Verbindung zum KMS-Server läuft die Aktivierung ab. Windows zeigt dann einen Hinweis und schränkt ggf. Funktionen ein. Lösung: VPN ins Firmennetz aufbauen, damit sich Windows neu aktivieren kann.

    Vertiefung CAL-Bedarf berechnen

    Die „Bäckerei Sonnenschein GmbH" betreibt einen Windows Server 2022. Folgende Nutzer greifen zu:

    • 8 Büromitarbeiter, jeder hat einen eigenen PC
    • 12 Bäcker in der Produktion, die sich 4 Produktions-Terminals teilen
    • 2 Fahrer, die jeweils ein Tablet nutzen (nur für Dateiabfragen)

    Aufgabe: 1. Berechne den günstigsten CAL-Bedarf für jeden Bereich 2. Erkläre, welchen CAL-Typ du für welchen Bereich wählst und warum 3. Wie viele CALs benötigt das Unternehmen insgesamt?

    Tipp

    Büro: 8 Personen, 8 Geräte → gleich teuer. Produktion: 12 Personen, 4 Geräte → welches ist günstiger? Fahrer: 2 Personen, 2 Geräte → gleich teuer, wähle pragmatisch.

    Lösung

    Büromitarbeiter: 8 Personen, 8 Geräte → gleich teuer → Wahl: User-CAL × 8 (oder Device-CAL × 8, macht keinen Unterschied)

    Produktionsbäcker: 12 Personen, 4 Terminals → 4 Geräte günstiger! → Wahl: Device-CAL × 4 (12 User-CALs wären unnötig teuer)

    Fahrer: 2 Personen, 2 Tablets → gleich teuer → Wahl: User-CAL × 2 (da Fahrer ggf. auch von anderen Geräten zugreifen könnten)

    Gesamtbedarf: 8 + 4 + 2 = 14 CALs

    Zum Vergleich: Nur User-CALs wären 8+12+2 = 22 CALs – 8 mehr.

    Herausforderung Lizenzierungskonzept erstellen

    Du bist IT-Verantwortlicher bei der „TechMedics GmbH" (Medizintechnik-Startup, 35 Mitarbeiter).

    Folgende Situation: - 25 feste Büromitarbeiter mit je eigenem PC (Windows) - 10 Außendienstmitarbeiter mit je einem Firmenlaptop (Windows), die selten im Büro sind - 1 Windows Server 2022 für Dateifreigaben und Active Directory - Buchhaltungssoftware mit NUL-Modell (3 Buchhalter) - Microsoft Office wird von allen 35 Mitarbeitern genutzt

    Erstelle ein vollständiges Lizenzierungskonzept mit folgenden Punkten:

    1. Welche CAL-Variante wählst du (User-CAL oder Device-CAL) und wie viele?
    2. Wie aktivierst du Windows auf den 35 Geräten – und warum?
    3. Welche Lizenzanzahl für Office und warum?
    4. Welche Risiken entstehen, wenn die Außendienstler lange nicht im Büronetz sind?
    Tipp

    Denke bei CALs an Außendienstler: Die haben einen Laptop, greifen aber auch vom Homeoffice oder Hotel aus zu. Bei KMS: Wie oft sind die Außendienstler im Firmennetz?

    Lösung

    1. CAL-Variante: User-CAL × 35, weil: - Büromitarbeiter: 25 Personen = 25 Geräte (gleich teuer, aber User-CAL ist flexibler) - Außendienstler: 10 Personen, die von verschiedenen Orten zugreifen → User-CAL zwingend sinnvoller

    Keine Device-CALs, weil Außendienstler auch von zuhause oder VPN zugreifen.

    2. Windows-Aktivierung: KMS-Aktivierung über den internen Windows Server. - 35 Geräte übersteigen die Mindestgrenze (25) für KMS - Zentrale Verwaltung spart Zeit - Risiko: Außendienstler müssen sich regelmäßig per VPN einloggen (alle 180 Tage)

    3. Office-Lizenzen: Microsoft 365 Business (User-Lizenz) × 35 - Jeder der 35 Mitarbeiter braucht eine Lizenz - User-Lizenz erlaubt Office auf mehreren Geräten (PC + Laptop)

    4. Risiken für Außendienstler: - KMS-Aktivierung läuft nach 180 Tagen ab → Windows zeigt Warnmeldung - Lösung: VPN-Pflicht alle 3 Monate oder Active Directory Domain Join mit Azure AD ermöglichen Onlineaktivierung
